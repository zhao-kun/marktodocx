marktodocx Architecture

Shared core + two runtime families — four hosts, one normalized DOCX contract

Shared Core Owns

  • • Canonical style + layout schemas
  • • Markdown parsing + syntax highlighting
  • • Image inlining + table/blockquote rules
  • • HTML → DOCX via html-to-docx
  • • DOM adapter + Mermaid fragment ABI
  • • Fixture-driven parity utilities

Runtime Family Split

  • • Browser family: native DOM + in-page Mermaid
  • • Node family: jsdom/linkedom + filesystem
  • • Puppeteer isolated to Mermaid helper only
  • • CLI / skill stay lean on the common path
  • • Fail fast if Mermaid helper is unavailable
  • • No whole-document browser launcher on Node

Parity Contract

  • • Normalized DOCX, not raw zip bytes
  • • Strip rsid, timestamps, revision noise
  • • Mermaid: SVG hash + declared extents
  • • Pinned versions via workspace overrides
  • • Manifest-driven golden corpus
  • • Runs on every PR, not only nightly
